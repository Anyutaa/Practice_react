# Project Practice — Gas Economy

Учебный мини-проект, состоящий из **Frontend** и **Backend** частей.
Приложение отображает прогнозные показатели газовой добычи (валовая/товарная добыча, конденсат, расход газа на СТН и др.) на горизонте **2026–2040 гг.** в виде редактируемой таблицы и интерактивных графиков.

При открытии страницы пользователь видит **7 показателей** со значениями по годам, полученных от backend-сервиса. Таблицу можно редактировать, добавлять и удалять строки, а изменения отправляются на сервер и сохраняются в JSON-файле.

---

## Описание возможностей

### Frontend
- Редактируемая таблица с фиксированными столбцами «Показатель» и «Единицы измерения»
- Колонки по годам строятся **динамически** на основе ключей объекта `meanings` — если в `data.json` загрузить набор с другим диапазоном лет (например, 2020–2050), таблица автоматически отрисует нужное количество столбцов
- Горизонтальный скролл по годам и фиксированная высота таблицы
- Добавление новой строки — у новых строк используется временный `tempId`, который после сохранения заменяется реальным `id` с сервера
- Удаление строки: клик по названию показателя выделяет строку целиком, иконка удаления убирает её
- Меню табов — при выборе показателя отображается соответствующий график значений по годам
- Построение графиков на Chart.js
- Кнопка «Сохранить» — батчевая отправка добавленных, изменённых и удалённых строк на сервер за один клик

### Backend
- CRUD-методы для работы с показателями (получение, добавление, изменение, удаление)
- Все операции изменения работают с **массивами** — за один запрос можно добавить/обновить/удалить несколько строк
- Чтение и запись данных в JSON-файл (`data.json`)
- REST API для взаимодействия с клиентом
- CORS включён для локальной разработки
- Обработка ошибок

## REST API

Базовый URL: `http://localhost:3001/data`

| Метод  | Endpoint        | Тело запроса                                | Описание                              |
|--------|-----------------|---------------------------------------------|---------------------------------------|
| GET    | `/data`         | —                                           | Получить все показатели               |
| POST   | `/data/add`     | `Array<Row>` (с `tempId`)                   | Добавить массив новых строк           |
| PATCH  | `/data/update`  | `Array<{ id, changes }>`                    | Обновить набор строк                  |
| DELETE | `/data/delete`  | `Array<number>` — массив `id`               | Удалить строки по `id`                |

### `GET /data`

**Ответ:**
```json
[
  {
    "id": 1,
    "name": "Газ (вал.)",
    "unit_name": "млн м3",
    "meanings": {
      "2026": 370,
      "2027": 500,
      "2028": 580,
      "2029": 590,
      "2030": 600
    }
  }
]
```

### `POST /data/add`

Добавление нескольких строк сразу. У новых строк есть временный `tempId`, по которому потом подменяются реальные `id` из ответа сервера.

**Запрос:**
```json
[
  {
    "tempId": "tmp_1",
    "name": "Новый показатель",
    "unit_name": "тыс.т",
    "meanings": { "2026": 0, "2027": 0, "2028": 0 }
  }
]
```

**Ответ:**
```json
{
  "idMap": {
    "tmp_1": 8
  }
}
```

### `PATCH /data/update`

Частичное обновление. Передаётся массив объектов с `id` строки и набором изменённых полей в `changes`. Поле `meanings` обновляется как объект «год → значение».

**Запрос:**
```json
[
  {
    "id": 1,
    "changes": {
      "name": "Газ (валовая добыча)",
      "unit_name": "млн м3",
      "meanings": { "2026": 380.5, "2027": 510 }
    }
  }
]
```

### `DELETE /data/delete`

Тело запроса — массив `id` для удаления (передаётся через `axios.delete(url, { data: [...] })`).

**Запрос:**
```json
[2, 5, 7]
```

## Формат хранимых данных (`backend/data.json`)

Файл содержит массив показателей. У каждого показателя есть `id`, `name`, `unit_name` и объект `meanings`, ключи которого — годы, а значения — числа.

Количество ключей в `meanings` не фиксировано: фронтенд строит колонки таблицы и точки на графике по тем годам, которые есть в файле. В стартовом наборе используются 7 показателей и горизонт 2026–2040 гг., но загрузить можно любой другой диапазон лет.

```json
[
  {
    "id": 1,
    "name": "Газ (вал.)",
    "unit_name": "млн м3",
    "meanings": {
      "2026": 370,
      "2027": 500,
      "2028": 580,
      "2029": 590,
      "2030": 600,
      "2031": 620,
      "2032": 620,
      "2033": 610,
      "2034": 620,
      "2035": 620,
      "2036": 580,
      "2037": 580,
      "2038": 580,
      "2039": 620,
      "2040": 620
    }
  }
]
```

## Логика «Сохранить» на фронтенде

Функция `saveData` в `src/api/jsonApi.js` выполняет три запроса последовательно:

1. **`saveAddedRows`** → `POST /data/add` — отправляет добавленные строки и подменяет `tempId` на реальные `id` через `idMap`.
2. **`saveEditedRows`** → `PATCH /data/update` — формирует payload вида `{ id, changes }`. Значения в `meanings` парсятся в число (с заменой запятой на точку, чтобы поддержать русскоязычный ввод вроде `12,5`); если строка не парсится — значение остаётся как есть.
3. **`deleteRows`** → `DELETE /data/delete` — отправляет массив удалённых `id`.

После успешного сохранения локальные стейты `addedRows`, `editedRows`, `deletedIds` очищаются.

## Возможные ошибки

1. **`ECONNREFUSED` при запросе с фронтенда** — backend не запущен. Проверьте, что сервис стартует на `http://localhost:3001`.
2. **CORS-ошибка в браузере** — в `backend/src/main.ts` должен вызываться `app.enableCors()` (уже включён в проекте).
3. **Изменения не сохраняются в файл** — проверьте права на запись в `backend/data.json`.
4. **`idMap` не приходит в ответе POST** — проверьте, что `app.service.ts` возвращает объект формата `{ idMap: { tempId: realId } }`.
5. **Запятые в числах теряются** — `saveEditedRows` заменяет `,` на `.` перед `parseFloat`. Если значение не должно быть числом, оно сохранится как строка.
